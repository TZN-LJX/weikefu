# 威科夫 ETH 合约学习与决策训练系统 SPEC

- 日期：2026-07-16
- 状态：需求已确认，等待实施计划
- 使用范围：面向单一用户使用，通过公开 GitHub Pages 静态托管
- 目标平台：手机与电脑响应式网页，可添加到手机桌面
- 核心标的：Binance U 本位永续 ETHUSDT

## 1. Problem Statement

用户已经阅读过一遍《威科夫操盘法》，但目前主要停留在“见过术语”的阶段，尚不能稳定地从陌生图表中识别市场背景、供需变化和可交易位置。

当前交易场景为 Binance ETHUSDT U 本位永续合约，常用 5 至 10 倍杠杆，计划以 4 小时判断背景、1 小时识别结构和入场，持仓约 1 至 3 天，同时学习做多与做空。当前交易结果总体亏损，主要问题是：

1. 无法稳定判断市场背景，容易把局部形态当作完整信号。
2. 缺少统一的分析和决策流程，每次交易判断标准不一致。
3. 仓位和杠杆偏大，尚未形成基于止损距离与账户风险的仓位纪律。
4. 原书 288 页和 61 页核心笔记不适合每天在手机上主动练习。
5. 普通阅读和选择题无法证明用户能将威科夫逻辑迁移到真实 ETH 行情。
6. 直接看历史答案容易产生后见之明，实时行情又容易诱发冲动交易。
7. 如果系统直接给信号，用户可能依赖工具，而非真正掌握方法。

产品需要把原书内容转化为一个私人使用、手机和电脑均适配的持续训练系统，使用户在约 90 天内逐步达到以下最低标准：

- 连续学习至少 30 天。
- 基础知识正确率达到 80% 以上。
- 陌生图表判断正确率达到 90% 以上。
- 能用供需和市场背景完整解释决策。
- 完成至少 30 笔遵守规则的内部模拟交易。

最终业务目标是改善实盘决策质量、控制风险并争取长期盈利，但产品不承诺收益，也不以短期盈亏作为唯一学习验收标准。

## 2. Goals

### 2.1 学习目标

- 将原书的概念、图解和分析顺序转化为可每日完成的主动训练。
- 让用户从“识别术语”进阶到“解释供需逻辑”和“处理陌生图表”。
- 通过间隔重复、错题和变式案例形成长期记忆。
- 通过费曼解释检验用户是否真正理解，而非只会选择答案。

### 2.2 交易行为目标

- 固化“背景 -> 价量证据 -> 结论 -> 行动 -> 失效条件”的分析顺序。
- 训练用户把“不交易”视为正常且合格的决策。
- 强制执行单笔账户风险不超过 1% 的模拟交易纪律。
- 让交易日志能够区分分析错误、执行错误和情绪错误。

### 2.3 持续使用目标

- 默认首页直接给出 15 至 20 分钟的今日任务。
- 降低选择成本，不要求用户每天自行安排学习内容。
- 以轻量闯关和阶段解锁保持进度感，不使用社区排名刺激交易频率。

## 3. Proposed Solution

产品定位为“威科夫学习与交易决策训练系统”，采用自适应闯关主线，并在每个基础模块结束后插入一段隐藏未来走势的 ETH 历史回放。

### 3.1 内容权威与适配规则

内容权威顺序为：

1. 《威科夫操盘法》原书。
2. 61 页核心笔记，作为复习提纲。
3. ETH 适配说明，作为市场差异解释。

适配规则：

- 保持原书术语、供需逻辑和分析顺序不变。
- ETH 适配层只更换案例并解释 24 小时交易、合约成交量等市场差异。
- 只使用价格、成交量和走势速度。
- ETH 训练使用 Binance ETHUSDT 永续合约自身的成交量。
- 原书内容与 ETH 适配说明必须在界面中明确标识。
- 公开站点不得包含原始 PDF、原书原图或大段原文。
- 公开内容只使用个人整理摘要、必要短引用和重新设计的原创图解。
- 完整 PDF、原书图片、大段原文、完整课程映射和私有案例允许进入本地私有学习包，但不得进入 GitHub 仓库或 GitHub Pages 构建产物。
- 私有学习包通过手机文件选择器导入，应用不得把包内容上传到 GitHub 或其他业务服务器。

### 3.2 课程结构

课程分为七个阶段：

1. 风险纪律与聪明钱看盘顺序。
2. 供需、背景、支撑阻力、努力与结果。
3. 吸筹与做多逻辑。
4. 派发与做空逻辑。
5. Spring、测试与失败案例。
6. ETH 历史行情综合回放。
7. 30 笔内部模拟交易。

吸筹和派发采用对照教学。课程不等到理论全部结束才进入实战；每完成一个基础模块，必须加入至少一段隐藏未来走势的 ETH 历史回放。

每个可教学知识单元必须包含：

- 一段短讲解。
- 对应的原书章节和页码。
- 一张带来源引用的原创交互图解。
- 一道基础检查题。
- 一道图表标注或证据选择题。
- 一个错误解释和对应纠正方式。

### 3.3 每日自适应学习

默认首页为“今日任务”。系统根据复习计划和薄弱点，生成约 15 至 20 分钟的任务：

1. 错题复习。
2. 新知识短讲解。
3. 基于原书概念的原创图解标注。
4. ETH 变式图训练。
5. 费曼解释。

单关目标时长约 5 分钟。用户可以查看闯关地图，但不需要自行决定每天先学什么。

案例按以下顺序开放：

1. 典型且答案清楚的案例。
2. 同一原则的变式案例。
3. 存在合理歧义的案例。
4. 形态失败案例。
5. 没有交易机会的案例。

### 3.4 掌握度与解锁

- 基础知识正确率用于衡量概念掌握度。
- 图表题同时评估结论和证据。
- 关卡解锁要求当前单元正确率达到 90%，并完成合格解释。
- 合理歧义案例不要求匹配唯一标签，但结论必须与背景、价量证据和失效条件自洽。
- 错误内容自动进入间隔复习队列，直到延迟复测通过。
- AI 评分不是唯一解锁依据；结构化必填证据和规则评分必须能够独立工作。

### 3.5 图表训练交互

手机默认使用纵屏分步引导，一次只要求完成一个判断：

1. 查看 4 小时市场背景。
2. 判断 1 小时所处阶段。
3. 比较上涨波和下跌波的价量强度。
4. 标记支撑、阻力、趋势线和威科夫事件。
5. 判断是否存在交易机会。
6. 给出入场、止损、目标和失效条件。
7. 说明为什么交易或为什么不交易。

需要精细标注时，用户可以按需展开横屏全图。系统不得在用户提交前显示未来 K 线、答案标签或暗示性评分。

所有正式判断只使用已经收盘的 K 线。未收盘 K 线可以显示，但必须明确标记为不可评分状态；第一版训练题默认不展示未收盘 K 线。

### 3.6 证据阶梯反馈

答错后不立即显示标准答案，而是：

1. 标出被遗漏或被错误解释的价量证据。
2. 询问证据对供需关系意味着什么。
3. 要求用户按固定结构重新解释：
   - 看到什么。
   - 说明什么。
   - 预期什么。
   - 如何行动。
   - 什么情况证明判断错误。
4. 解释达到要求后，显示参考分析。
5. 最后揭示后续走势，并把错误加入复习计划。

第一版使用文字输入。语音讲解安排在第二阶段。

### 3.7 内部模拟交易

内部模拟账户初始资金为 1,000 USDT。每笔模拟交易记录：

- 方向。
- 入场价。
- 止损价。
- 目标价。
- 仓位名义价值。
- 所需保证金。
- 账户风险金额和比例。
- 计划盈亏比。
- 实际结果和 R 倍数。

第一版不精确模拟手续费、资金费率、滑点和强平过程。

风险计算规则：

- 风险金额 = 当前模拟账户权益 x 1%。
- 止损距离比例 = abs(入场价 - 止损价) / 入场价。
- 风险约束仓位 = 风险金额 / 止损距离比例。
- 保证金约束仓位 = 可用模拟权益 x 用户选择的杠杆倍数。
- 最大仓位名义价值 = min(风险约束仓位, 保证金约束仓位)。
- 计划 R 倍数 = abs(目标价 - 入场价) / abs(入场价 - 止损价)。
- 做多时止损必须低于入场价；做空时止损必须高于入场价。
- 杠杆影响保证金占用和可建立的最大名义仓位，但不提高最大可接受风险金额。

以下任一条件不满足时，阻止内部模拟下单：

- 已判断 4 小时背景。
- 已判断 1 小时结构和事件。
- 已提供供需证据。
- 已提供入场后的确认条件。
- 已提供失效条件和止损。
- 单笔账户风险不超过 1%。
- 已明确检查“不交易”理由。

系统只能阻止内部模拟订单，不能阻止用户在外部 Binance 应用自行下单。

### 3.8 交易日志

每笔模拟交易记录：

- 4 小时背景。
- 1 小时阶段和威科夫事件。
- 关键价量证据。
- 入场、止损、目标、仓位和账户风险。
- 计划、确认条件和失效条件。
- 是否存在不交易理由。
- 情绪和冲动程度。
- 是否违反规则。
- 出场原因、实际 R 倍数和复盘解释。

日志使用分步短表单，避免在手机上一次展示全部字段。复盘必须区分：

- 分析错误。
- 仓位或风险错误。
- 执行错误。
- 情绪和纪律错误。
- 合格决策但结果亏损。

### 3.9 AI 角色

AI 仅承担：

- 苏格拉底式追问。
- 费曼解释评分。
- 图表分析陪练。
- 交易日志复盘教练。

约束：

- 用户必须先提交自己的判断，AI 才能反馈。
- AI 不直接输出最终交易信号。
- AI 不得批准违反风险规则的模拟订单。
- AI 反馈应引用用户选择的证据和当前课程规则。
- AI 请求只发送当前题目所需的最小内容片段，不得发送完整 PDF 或整个私有学习包。
- 没有 AI Key 或 AI 服务失败时，核心课程必须继续可用。

### 3.10 分阶段交付

第一阶段包括：

- 课程与闯关地图。
- 今日自适应任务。
- 原创教学图解和人工筛选的 ETH 历史回放。
- 文字费曼训练。
- 错题与间隔复习。
- 内部模拟交易和交易日志。
- 使用隐藏未来走势的历史回放完成第一阶段 30 笔模拟交易。
- 本地保存及导入 / 导出备份。
- 本地私有学习包导入、校验、替换和删除。
- 用户自带 Key 的可选 AI 反馈。

第二阶段包括：

- Binance 公开历史数据下载。
- Binance 实时行情。
- 4 小时与 1 小时联动。
- 收盘 K 线训练。
- 用户先判断、系统后核对。
- 语音费曼讲解。

## 4. Information Architecture

### 4.1 主导航

- 今日：自适应任务和待复习内容。
- 闯关：七阶段课程地图与解锁状态。
- 训练：原创教学图解、ETH 回放和模拟交易。
- 复盘：错题、费曼解释、交易日志和掌握度。

### 4.2 首页

首页按以下优先级显示：

1. 当前阶段和总体进度。
2. 今日预计学习时间。
3. 错题复习任务。
4. 新知识任务。
5. ETH 隐藏回放任务。
6. 单一“开始今日训练”命令。

实时行情不得成为第一屏主内容。

## 5. Component Boundaries

### 5.1 Web App Shell

负责响应式布局、导航、桌面快捷方式 / Web App Manifest 和应用内待办。

### 5.2 Content Engine

负责课程、知识单元、原书引用、图解、题目、参考解释和内容版本。

### 5.3 Learning Engine

负责掌握度、关卡解锁、错题、间隔重复、今日任务和学习统计。

### 5.4 Chart Replay Engine

负责 4 小时 / 1 小时 K 线展示、未来数据遮蔽、图表标注、逐步揭示和案例评分。

### 5.5 Simulation Engine

负责模拟账户、仓位计算、风险限制、订单状态、盈亏和 R 倍数。

### 5.6 Journal Engine

负责交易计划、情绪记录、规则违规、出场复盘和错误归类。

### 5.7 AI Adapter

负责兼容接口配置、请求超时、结构化提示、反馈解析和无 AI 降级。

### 5.8 Local Data Layer

负责本地持久化、数据版本升级、备份导出、备份导入和冲突确认。

### 5.9 Private Content Pack Importer

负责读取 `.wkf` 私有学习包、验证清单和文件校验值、检查浏览器存储空间、写入本地数据库、替换旧版本和删除私有内容。导入过程不得产生网络上传。

### 5.10 Binance Market Adapter

第二阶段启用。只负责公开 ETHUSDT 永续市场数据，不接受账户或订单权限。

## 6. Core Data Entities

- ContentUnit：知识单元、来源、解释、图解和掌握要求。
- Exercise：题型、标准证据、可接受结论和评分规则。
- MarketCase：标的、周期、可见区间、隐藏区间、人工标签和来源。
- Attempt：用户答案、所选证据、解释、评分和错误类型。
- Mastery：知识点掌握度、连续正确次数和解锁状态。
- ReviewSchedule：复习时间、原因和完成状态。
- SimulationAccount：初始权益、当前权益和统计。
- SimulationTrade：计划、风险、订单结果和 R 倍数。
- JournalEntry：背景、证据、情绪、违规和复盘。
- Settings：界面、时区、AI 接口和本地偏好。
- ContentPackManifest：包标识、版本、内容结构版本、最低应用版本、来源指纹、资源索引和校验值。
- ContentAsset：只存在于本地浏览器中的 PDF、图片、原文、课程映射和私有案例资源。

## 7. Data Flows

### 7.1 每日学习

1. Learning Engine 读取掌握度和到期复习项。
2. 生成错题、新知识、图解和 ETH 回放组合。
3. 用户完成任务并提交证据和解释。
4. 规则评分先执行，AI 反馈按需执行。
5. 更新掌握度、复习计划和闯关进度。

### 7.2 历史回放

1. Chart Replay Engine 只载入截止点之前的数据。
2. 用户按步骤完成背景、结构、证据和行动判断。
3. 提交后执行规则评分和证据检查。
4. 错误时进入证据阶梯反馈。
5. 解释合格后逐步揭示未来走势。

### 7.3 模拟交易

1. 用户完成交易前检查。
2. Simulation Engine 计算风险、仓位和计划 R 倍数。
3. 不合格计划被阻止并返回具体缺失项。
4. 合格计划进入历史回放模拟。
5. 交易结束后创建 JournalEntry 并更新统计。

### 7.4 AI 反馈

1. 用户先提交结构化判断。
2. 系统只发送当前案例、用户答案和适用课程规则。
3. AI 返回问题、证据引用和解释评价。
4. 解析失败时显示可重试状态，不修改规则评分结果。

### 7.5 私有学习包导入

1. 用户通过手机文件选择器选择一个 `.wkf` 文件。
2. Importer 在本地读取包清单，不发送网络请求。
3. 校验包版本、最低应用版本、资源索引和校验值。
4. 检查浏览器可用存储空间。
5. 用户确认后写入本地数据库并设为当前活动包。
6. 更新包时保持稳定内容 ID 对应的学习进度；不兼容内容必须明确提示。
7. 未导入有效学习包时，只开放设置、导入和帮助页面。

## 8. Technical Constraints

### 8.1 平台

- 第一版为部署在 GitHub Pages 的公开响应式静态网页。
- 手机宽度从 360px 开始适配。
- 主要手机验收环境为华为 Mate 80、HarmonyOS 6.1.0、华为浏览器 6.1.6.305。
- 辅助支持 Android Chrome、iPhone Safari、桌面 Chrome 和 Edge。
- 普通学习使用纵屏，精细图表支持横屏全屏。
- 华为浏览器必须支持通过 HTTPS 网址使用并添加到桌面；不要求课程离线使用。
- 图表使用成熟的金融图表库，不自行实现缩放和拖动内核。

### 8.2 静态部署与本地数据

- 第一版不要求长期运行的业务服务器。
- 代码和允许公开的课程资源部署到公开 GitHub 仓库 `TZN-LJX/weikefu` 的 GitHub Pages。
- 课程、题库、进度、日志和历史案例保存在浏览器本地。
- 结构化学习数据使用适合大量记录的浏览器本地数据库。
- 第一版不提供离线课程或离线启动，打开和刷新网站需要网络连接。
- AI、实时行情和首次下载市场数据同样需要网络连接。
- `APIKEY.txt`、原始 PDF、环境变量文件和用户备份不得进入公开仓库或构建产物。
- `.wkf` 私有学习包不得进入公开仓库或构建产物。
- 网站从网络加载后，可以读取已经导入本地数据库的私有内容；这不代表网站支持离线启动。

### 8.3 市场数据

- 标的固定为 Binance ETHUSDT U 本位永续。
- 只使用价格、成交量和走势速度。
- 多周期固定为 4 小时背景和 1 小时结构 / 入场。
- 正式评分只依据已收盘 K 线。
- 时间内部统一存储，界面按用户本地时区显示。
- 数据陈旧或连接中断时必须显示明确状态，不得伪装为实时数据。

### 8.4 Binance 权限

- 不要求或保存 Binance API Key。
- 不读取真实持仓、订单或账户资金。
- 不发送、修改或取消真实订单。
- 仅使用 Binance 公开市场数据。

### 8.5 本地数据与备份

- 第一版免登录。
- 必须支持版本化的导出和导入。
- 导入前验证结构和版本。
- 覆盖数据前必须确认。
- AI API Key 不得进入备份文件。
- 普通学习进度备份不包含私有学习包及包内出版物内容；换设备时需要单独重新导入 `.wkf` 文件。
- 数据升级不得无提示丢失学习记录。

### 8.6 AI 接口和安全

用户配置兼容接口的 API 地址、模型名称和 API Key。

- Key 只保存在用户浏览器中。
- Key 不得写入日志、学习备份或交易日志。
- 请求只能发送给用户配置的服务地址。
- 如果服务商不允许浏览器直接调用，第一版停用 AI，不增加付费代理服务器。
- 没有 Key、余额不足、超时或异常时，固定规则学习流程继续运行。

### 8.7 确定性逻辑

以下逻辑不得交给 AI 决定：

- 知识题正确率。
- 关卡解锁。
- 间隔复习日期。
- 仓位与风险计算。
- 止损距离、盈亏和 R 倍数。
- 单笔风险是否超过 1%。
- 模拟订单是否满足强制检查项。

### 8.8 性能、网络与可用性

- 今日首页不得等待行情或 AI 加载后才能使用。
- 课程、图解和图表数据按需加载。
- 触摸目标适合手机单手操作。
- 文本、按钮、图表和反馈不得在支持的视口中重叠或裁切。
- 第一版提醒只使用应用内待办，不依赖系统推送。
- 网络中断时明确提示需要重新联网，不承诺离线继续学习。

## 9. Error Handling

### 9.1 AI 服务失败

- 保留用户已经输入的解释。
- 显示失败原因和可重试操作。
- 继续执行规则评分和固定证据反馈。
- 不因 AI 失败扣除掌握度或阻止非 AI 课程。

### 9.2 市场数据失败

- 显示离线、陈旧或缺失状态。
- 停止实时评分。
- 不使用缓存旧数据冒充新数据。
- 当前页面已经载入的历史案例可以继续完成；刷新或重新打开网站需要网络连接。

### 9.3 本地存储失败

- 保存失败时立即提示用户导出备份。
- 不显示虚假的“已保存”状态。
- 导入损坏或版本不兼容的备份时，不覆盖当前数据。

### 9.4 内容或评分异常

- 缺少来源、证据标签或隐藏区间的案例不得进入正式题库。
- 评分规则异常时保留答题记录，并将该题标记为不可计分。
- AI 结论与确定性规则冲突时，以确定性规则和人工审核内容为准。

### 9.5 模拟订单异常

- 无有效止损、方向与止损关系错误、风险计算无效时拒绝提交。
- 缺少关键检查项时逐项显示原因。
- 重复提交不得创建重复模拟订单。

### 9.6 私有学习包异常

- 文件格式、版本或校验值无效时拒绝导入，不覆盖当前内容。
- 浏览器存储空间不足时，在写入前停止并显示所需空间。
- 导入中断时清理未完成数据，不产生半导入状态。
- 替换包前必须确认；内容 ID 不兼容时说明可能受影响的学习进度。
- 应用不得通过日志、AI 请求或网络接口发送整个私有学习包。

## 10. Acceptance Criteria

### 10.1 内容

- 原书五章核心方法均有课程映射。
- 每个知识单元都有来源、练习和错误纠正方式。
- 原书内容与 ETH 适配内容标识清楚。
- 课程和 ETH 案例完成过人工审核。

### 10.2 学习流程

- 首页能根据掌握度生成今日任务。
- 错题进入间隔复习并能在延迟复测后退出。
- 解锁规则同时检查正确率和解释完成度。
- 没有 AI Key 时仍能完成核心学习闭环。

### 10.3 图表回放

- 用户提交前无法查看未来 K 线。
- 4 小时与 1 小时数据正确对齐。
- 手机纵屏分步训练和横屏标注均可操作。
- 合理歧义案例支持基于证据的多种答案。

### 10.4 模拟交易

- 风险金额、仓位名义价值和 R 倍数计算正确。
- 单笔账户风险超过 1% 时模拟订单无法提交。
- 做多 / 做空止损方向错误时模拟订单无法提交。
- 合格亏损可以被记录为“过程正确、结果亏损”。

### 10.5 数据可靠性

- 导出文件可在新浏览器中恢复学习进度、错题、模拟交易和日志。
- API Key 不包含在导出文件中。
- 数据接口失败时不会显示虚假实时状态。
- 数据结构升级不会无提示丢失历史记录。

### 10.6 私有学习包

- 华为浏览器能够从手机文件中导入使用真实两份 PDF 生成的 `.wkf` 学习包。
- 导入过程中没有私有包内容上传到网络。
- 导入后可以查看包内 PDF、图片、原文、课程和私有案例。
- 刷新或更新 GitHub Pages 后，本地学习包仍然可用。
- 包格式损坏、版本不兼容或存储不足时不会破坏已有内容和进度。
- 删除包后清除包内出版物内容，但保留前必须明确告知进度数据的处理方式。

### 10.7 响应式体验

- 360px 手机宽度下无关键内容重叠或裁切。
- 华为 Mate 80 的华为浏览器 6.1.6.305 能通过网址完成核心流程并添加到桌面。
- Android Chrome、iPhone Safari、桌面 Chrome / Edge 能完成核心流程。
- 图表缩放、拖动、标注和横竖屏切换不造成答案泄露或数据错位。

## 11. Success Metrics

90 天学习目标：

- 连续学习至少 30 天。
- 基础知识正确率达到 80% 以上。
- 陌生图表判断正确率达到 90% 以上。
- 能完整说明背景、证据、行动和失效条件。
- 完成至少 30 笔遵守规则的内部模拟交易。

辅助指标：

- 今日任务完成率。
- 延迟复测通过率。
- “不交易”决策的合理率。
- 模拟交易规则遵守率。
- 违反仓位、止损或分析顺序的次数。
- 模拟交易 R 倍数分布和最大回撤。

实盘盈利只作为长期观察指标，不作为产品保证或短期课程解锁条件。

## 12. Non-goals

### 12.1 产品始终不做

1. 不承诺胜率、收益或一定能够赚钱。
2. 不把短期实盘盈亏作为唯一学习成效。
3. 不替用户作出最终交易决定。
4. 不提供无解释的多空喊单、精确点位预测或“必胜机会”。
5. 不自动创建、修改或取消 Binance 真实订单。
6. 不连接 Binance 账户，不读取真实持仓和资金。
7. 不允许 AI 绕过风险规则批准模拟订单。
8. 不把 AI 判断当作威科夫方法的权威来源。
9. 不为了提高题目确定性而篡改原书定义。
10. 不保证每段行情都存在交易机会。
11. 不发展成 TradingView 替代品或通用金融图表平台。
12. 不加入自动参数优化、量化策略搜索或黑箱回测。
13. 不加入跟单、代客交易或资产管理。
14. 不上传或分发原始 PDF、原书原图、大段原文或其他未经授权的出版物内容。

### 12.2 第一版不做

1. ETH 以外的币种和交易对。
2. 4 小时与 1 小时之外的主要交易周期。
3. MACD、RSI、均线策略等其他指标体系。
4. 持仓量、资金费率、清算数据、链上数据、新闻和基本面分析。
5. 实时自动扫描威科夫结构或主动推送交易机会。
6. 实时行情和实时判断。
7. Binance 手续费、资金费率、滑点和强平机制的完整仿真。
8. 真实账户盈亏统计、税务和资产组合管理。
9. 手机号、微信或邮箱账号体系。
10. 云端同步和多人共享。
11. 社区、排行榜、公开学习动态和比赛。
12. 订阅、支付、会员和商业化功能。
13. 原生 Android、iPhone 应用和微信小程序。
14. 语音转写与语音费曼训练。
15. PWA 系统推送、微信或邮件提醒。
16. 课程离线缓存和离线启动。
17. 全自动从 PDF 生成并发布课程、题目和标准答案。
18. 完全自由的 AI 聊天入口。
19. 精确模拟真实交易撮合。
20. 自动阻止用户在外部 Binance 应用中自行下单。
21. 多用户账号、公开学习社区或面向其他用户运营。
22. 将私有学习包上传到业务服务器、GitHub 仓库或 GitHub Pages。
23. 第一版不提供私有学习包密码加密；文件保密依赖用户私下保存和传输。
